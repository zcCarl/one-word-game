# Roguelike Turn-Based Maze Brief

## One-Line Request

做一个肉鸽类回合制游戏，参考《不思议迷宫》那种探索、翻格、战斗、成长和随机事件体验。

## Product Intent

玩家在一个未知迷宫中逐格探索。每次行动推进回合，翻开地块后可能遇到怪物、资源、商店、事件、宝箱或楼梯。玩家需要在生命、能量、金币、装备和技能之间做取舍，尽量深入迷宫并击败本层 Boss。

## MVP Name

`迷宫回合 Rogue Demo`

## Target Experience

- 10 分钟内能理解规则并完成一局。
- 每一步都有风险和奖励。
- 每局地图、怪物、奖励和事件不同。
- 玩家能感到“我可以再来一局，因为刚才的构筑还差一点”。

## MVP Scope

- 单人离线 demo。
- 1 个可玩的随机楼层，包含 6x6 或 7x7 翻格地图。
- 回合制行动：移动/翻格/攻击/使用技能。
- 3 种怪物、1 个 Boss、3 种事件、3 种道具/装备、1 个商店。
- 基础属性：HP、Attack、Defense、Gold、Energy、Level、XP。
- 核心循环：探索 -> 遭遇 -> 战斗/事件 -> 奖励 -> 成长 -> 找到楼梯/Boss -> 胜负结算。
- 开始菜单、HUD、日志、结算、重开。

## Non-MVP

- 多楼层完整长线进度。
- 联网、存档、账号。
- 复杂职业系统。
- 大量美术资产。
- 复杂动画和全套音频，仅保留关键反馈。

## Key Risks

- 只做成“点格子看文字”，缺少策略和反馈。
- 随机事件没有权衡，导致肉鸽感不足。
- 战斗公式过于平，玩家选择不重要。
- UI 信息不清楚，玩家不知道下一步该做什么。

## Success Criteria

- 首屏进入就是可玩的游戏，不是说明页。
- 玩家能完成一局并看到胜利或失败。
- 随机性影响路线、资源和构筑。
- QA 能按验收文档复现核心流程。
