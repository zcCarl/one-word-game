# owg-ui-implementation

## Use When

The agent implements the playable UI.

## Required Inputs

- Game state model
- Generated art assets
- Acceptance checklist

## Workflow

1. Build board, HUD, action panel, log, modals, and result screens.
2. Make every tile state visually distinct.
3. Disable invalid actions.
4. Keep text readable on desktop and narrow viewport.
5. Add visual feedback for reveal, hit, reward, level up, victory, and defeat.

## Output Standard

- UI components
- Responsive checks
- Screenshot notes
- Accessibility/readability concerns

## Hard Rules

- Do not create a landing page instead of the game.
- Do not let text overlap.
- Do not rely only on color for critical state.
