# Roguelike Turn-Based Maze Acceptance Checklist

## Launch

- [ ] App starts from documented command.
- [ ] First screen is playable or one click away from playable.
- [ ] No console errors on initial load.

## Core Loop

- [ ] Player can reveal adjacent tiles.
- [ ] Revealed tiles trigger encounters.
- [ ] Movement/reveal/attack advances turns.
- [ ] Combat resolves with player and enemy damage.
- [ ] Rewards update player stats.
- [ ] Level up occurs after enough XP.
- [ ] Boss encounter can be reached.
- [ ] Boss defeat triggers victory.
- [ ] HP reaching 0 triggers defeat.
- [ ] Restart creates a fresh run.

## Content

- [ ] At least 3 monster types.
- [ ] At least 3 events.
- [ ] At least 3 item/equipment rewards.
- [ ] At least 1 shop.
- [ ] At least 1 active skill.

## UI

- [ ] HUD shows HP, Attack, Defense, Gold, Energy, Level, XP, Turn.
- [ ] Board clearly distinguishes hidden, available, revealed, current, enemy, treasure, event, shop, boss.
- [ ] Action buttons disable when unusable.
- [ ] Event/shop choices are readable.
- [ ] Log records meaningful actions.
- [ ] Victory/defeat screens show score and restart.
- [ ] Text does not overlap on desktop and narrow viewport.

## Game Feel

- [ ] Important actions have visual feedback.
- [ ] Damage/reward changes are obvious.
- [ ] Visual theme reads as dungeon roguelike, not default UI.
- [ ] Audio plan or basic audio feedback exists.

## Balance

- [ ] Player can win with reasonable decisions.
- [ ] Player can lose through bad decisions.
- [ ] Skill and shop choices matter.
- [ ] Random generation does not produce unwinnable disconnected maps.
